# 차량용 LLM
------------

## Prepare
1. 차량 선택
2. 차량에 대한 문서
3. 실 사용자의 부품 교체
4. 실 사용자의 부품 이상
5. 기름/ 차량에 필요한 소모품 충전

## Process
1. 주제
2. 배경
3. 배경 소개 (이유, 진행 방향)
4. 아키텍쳐
5. 서비스 구현
6. 시행착오
7. 코드 설명
8. UI 설명
9. 최종 결론 (향후 적용 방안, 한계점, 개선점, 느낀 점)
- 한계점: 한 번 구동하는데 시간이 너무 오래걸려 디버깅이 힘들다.
- 개선점: 차종마다 기록에 남도록
